## Splitly (정산 + 가계부)

### 현황

- Phase 10 완료 (78개+ 언어, 보안 감사, i18n QA)
- Phase 11 세션 4까지 완료 (스토어 에셋, IAP, 스크린샷)
- Phase 12 진행 중 (기술 부채 + 기능 고도화 + 자동화 인프라)
- 스토어 배포 홀딩 (사업자등록 대기)

### 기술 스택

KMP + Compose Multiplatform, Room v5, Firebase, Koin 4.1.1, Kotlin 2.3.0, Next.js (웹)

### 수익 모델

```
무료: 기본 정산, 가계부, 2개 그룹
Premium ($2.99/월 or $19.99/년):
  ∙ 무제한 그룹
  ∙ AI 영수증 OCR
  ∙ AI 인사이트
  ∙ PDF/CSV 내보내기
  ∙ 광고 제거
Premium+ ($5.99/월 or $39.99/년):
  ∙ Premium 전체 +
  ∙ 환율 실시간
  ∙ 반복 정산 고급
  ∙ 우선 지원
AdMob: 무료 사용자에게 배너 광고
```

### 출시 후 로드맵

- ASO 최적화 (키워드, 스크린샷 A/B)
- 사용자 피드백 반영 (빠른 핫픽스)
- 크로스프로모션 (BetOnMe 출시 시 상호 홍보)
- Phase 13+: 소셜 기능 (친구 초대, 그룹 공유)

## BetOnMe (습관 내기) — 다음 개발 대상

### 왜 BetOnMe인가

| 기준 | BetOnMe | NoSpend | SubShare |
|------|---------|---------|----------|
| 경쟁자 수 | **1개 (Stickk)** | 10+ | 5+ |
| 구독 모델 적합성 | **높음** (반복 사용) | 중간 | 중간 |
| 글로벌 시장 | **전 세계** (습관은 보편적) | 전 세계 | 문화 의존적 |
| Splitly 모듈 재사용 | **65%** | 70% | 60% |
| 차별화 용이성 | **높음** (소셜 내기) | 낮음 | 중간 |
| 바이럴 가능성 | **높음** (친구와 내기) | 낮음 | 중간 |

### 컨셉

"친구와 돈을 걸고 습관 형성" — 운동, 공부, 금연 등 목표를 설정하고, 실패 시 벌금(자선단체 기부 또는 친구에게 전달). 성공 시 보상 + 스트릭 + 리더보드.

### 핵심 기능

```
[무료]
∙ 습관 목표 설정 (3개까지)
∙ 일일 체크인 (사진/텍스트 인증)
∙ 기본 통계 (스트릭, 완료율)
∙ 1:1 내기 (1개까지)

[Premium — $3.99/월 or $29.99/년]
∙ 무제한 목표
∙ 무제한 내기
∙ 그룹 챌린지
∙ AI 코칭 (Gemini)
∙ 상세 통계 + 인사이트
∙ 커스텀 벌금/보상
∙ 광고 제거
```

### 기술 계획

```
∙ 아키텍처: Clean Architecture + MVI (신규 패턴)
∙ 모듈: moon-iap, moon-auth, moon-premium, moon-admob 재사용
∙ 다국어: moon-i18n 하이브리드 (78개+)
∙ 백엔드: Firebase (Auth + Firestore + FCM + Functions)
∙ AI: Gemini API (코칭 기능)
∙ 결제 연동: Stripe 또는 Firebase Functions (벌금 전달)
∙ 예상 기간: 8-10주 (모듈 재사용 덕분)
```

### Stickk 대비 차별화

```
Stickk의 약점:
∙ 구식 UI (2008년 스타트업)
∙ 모바일 앱 UX 열악
∙ 다국어 미지원
∙ 소셜 기능 부족

BetOnMe 차별점:
∙ 현대적 Material 3 UI (Compose Multiplatform)
∙ 78개+ 언어 지원
∙ 소셜 내기 (친구/그룹)
∙ AI 코칭
∙ 사진 인증 시스템
∙ Android + iOS + Web 동시 지원
```

## NoSpend (지출 관리)

### 컨셉

"No-Spend Challenge" — 설정한 카테고리에서 일정 기간 지출 0원 도전. 게이미피케이션으로 동기 부여.

### Go/No-Go 판단: **대기**

경쟁이 많지만, Splitly 가계부 기능의 확장 버전으로 모듈 재사용률이 높아 빠르게 만들 수 있음. BetOnMe 이후 시장 반응 보고 결정.

## SubShare (구독 공유)

### 컨셉

넷플릭스, 유튜브 등 구독 서비스를 친구/가족과 비용 분할 관리.

### Go/No-Go 판단: **대기**

문화적 차이가 크고(구독 공유 허용 여부가 국가/서비스별 다름), 법적 리스크 검토 필요. 3번째 이후 검토.

## 신규 앱 Go/No-Go 체크리스트

새 앱 아이디어가 나올 때마다 이 체크리스트로 평가:

```
[시장 — 3개 중 2개 이상 통과 필수]
☐ TAM (Total Addressable Market) > 1M 사용자?
☐ 직접 경쟁자 5개 미만?
☐ 글로벌 시장 접근 가능? (문화 의존성 낮음?)

[수익 — 2개 중 1개 이상]
☐ 구독 모델 적합? (반복 사용 패턴?)
☐ 무료 사용자도 가치를 느끼는 Freemium 설계 가능?

[개발 — 3개 중 2개 이상]
☐ 기존 모듈 재사용 40% 이상?
☐ MVP 10주 이내 가능?
☐ 1인 개발/운영 가능? (서버 부담 적음?)

[차별화 — 2개 중 1개 이상]
☐ 기존 경쟁자 대비 명확한 강점?
☐ 바이럴/소셜 요소 존재?

[최종 판단]
∙ 7/10 이상 통과 → Go
∙ 5-6/10 → 추가 리서치 후 결정
∙ 4/10 이하 → No-Go
```

## 크로스 프로모션 전략

```
앱 2개 이상 출시 후:

[인앱 크로스 프로모션]
∙ Splitly 설정 → "MoonDeveloper의 다른 앱" 섹션
∙ BetOnMe 온보딩 → "Splitly로 정산도 관리하세요" 배너
∙ 비침습적: 설정 페이지 또는 About 화면에 배치

[스토어 최적화]
∙ 개발자 페이지에서 전체 앱 노출
∙ 앱 설명에 "MoonDeveloper 시리즈" 언급
∙ 유사 키워드 교차 최적화

[공통 계정]
∙ Firebase Auth 공유 → 한 번 가입이면 다른 앱도 자동 로그인
∙ 프리미엄 번들 할인 (장기 목표)
```
