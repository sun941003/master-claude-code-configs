# MoonDeveloper — Monetization

> 수익화 전략, 가격 프레임워크, 투자 기준.
> 핵심: "무료 사용자 경험을 해치지 않으면서 자연스러운 전환"

---

## 수익 모델 프레임워크

### 3층 수익 구조 (모든 앱 공통)

```
Layer 1: AdMob (무료 사용자)
  ∙ 배너 광고 (하단, 비침습적)
  ∙ 전면 광고 (자연스러운 전환점에서만, 3회/시간 제한)
  ∙ 보상형 광고 (선택적 — 프리미엄 기능 1회 체험)
  ∙ 예상: $0.5-2 / DAU 1000

Layer 2: Freemium 구독 (핵심 수익)
  ∙ 무료 → 충분히 유용하게 (리텐션 확보)
  ∙ 프리미엄 → 파워 유저에게 진짜 가치 제공
  ∙ 연간 구독 할인 (월 대비 30-40% 할인)
  ∙ 전환율 목표: 무료 → 유료 3-5%

Layer 3: 확장 (장기)
  ∙ 프리미엄 번들 (여러 앱 할인)
  ∙ 오픈소스 스폰서십
  ∙ 기업용 라이센스 (B2B, 장기 목표)
```

## 가격 프레임워크

### 원칙

```
1. 글로벌 단일 가격 (USD 기준, 스토어가 환율 자동 적용)
2. 연간 구독이 메인 — 월간은 "비싸게 느끼게" 설정
3. 프리미엄 가치는 구체적 수치로 전달
   ✕ "더 많은 기능을 즐기세요"
   ✓ "AI가 영수증을 3초 만에 분석합니다" 
4. 무료 제한은 사용 불가가 아닌 "불편함" 수준
   ✕ 기능 잠금 (답답함 → 이탈)
   ✓ 광고 + 제한된 횟수 (체험 → 전환)
```

### 가격 테이블

| 앱 | 무료 | 월간 | 연간 | 연간 할인율 |
|----|------|------|------|-----------|
| Splitly | 기본 정산 + 2그룹 + 광고 | $2.99 | $19.99 | 44% |
| Splitly+ | — | $5.99 | $39.99 | 44% |
| BetOnMe | 3목표 + 1내기 + 광고 | $3.99 | $29.99 | 37% |

### Paywall UX 원칙

```
[타이밍 — 언제 보여줄까]
∙ 첫 사용 시: 보여주지 않음 (온보딩 → 핵심 기능 경험 후)
∙ 프리미엄 기능 터치 시: 가치를 먼저 보여주고 → Paywall
∙ 무료 한도 도달 시: "N회 남았습니다" → 0회 시 Paywall
∙ 자연스러운 성공 순간: 정산 완료 후 "PDF로 저장하시겠어요?"

[디자인]
∙ 기능 비교표 (무료 vs 프리미엄)
∙ 연간 구독 기본 선택 (월간보다 눈에 덜 띄게)
∙ "7일 무료 체험" 시작 버튼
∙ 닫기 버튼은 명확하게 (사용자 짜증 방지)
∙ 소셜 프루프: "10,000+ 사용자가 프리미엄"

[안티패턴 — 하지 말 것]
✕ 앱 시작 시 즉시 Paywall
✕ 닫기 버튼 숨기기
✕ 다크 패턴 (실수로 구독하게 유도)
✕ 무료 기능 갑자기 잠금
```

## 수익 예측

### 보수적 시나리오

```
[Splitly — 출시 6개월 후]
MAU: 5,000
유료 전환: 3% = 150명
구독 ARPU: $2.50/월 (월간+연간 혼합)
구독 수익: $375/월
AdMob: DAU 1,500 × $1/1000 = $45/월
합계: ~$420/월

[BetOnMe — 출시 6개월 후]
MAU: 3,000
유료 전환: 4% = 120명 (구독 모델이 더 적합)
구독 ARPU: $3.00/월
구독 수익: $360/월
AdMob: DAU 1,000 × $1/1000 = $30/월
합계: ~$390/월

[통합 6개월]
~$810/월
```

### 낙관적 시나리오

```
[Splitly — 출시 12개월 후]
MAU: 20,000 / 유료 5% = 1,000명
합계: ~$2,700/월

[BetOnMe — 출시 6개월 후]
MAU: 10,000 / 유료 5% = 500명
합계: ~$1,650/월

[통합 12개월]
~$4,350/월
```

## ASO (App Store Optimization)

### 기본 전략

```
[키워드]
∙ 각 언어별 로컬 키워드 리서치
∙ 경쟁 앱 키워드 분석
∙ 롱테일 키워드 공략 (경쟁 낮은 구체적 키워드)

[스크린샷]
∙ 첫 2장이 90% 결정 — 핵심 가치를 즉시 전달
∙ 텍스트 오버레이: 해당 언어로 번역
∙ A/B 테스트 (Google Play Experiments)

[설명]
∙ 첫 3줄에 핵심 가치 + 키워드 집중
∙ 기능 나열보다 사용자 혜택 중심
∙ 78개+ 언어 로컬라이즈

[리뷰 관리]
∙ InAppReview 적절한 타이밍에 요청
∙ 부정 리뷰 즉시 대응 (24시간 내)
∙ 반복 피드백 → 기능 개선에 반영
```

### 출시 초기 부스트

```
∙ Product Hunt 등록
∙ Reddit (r/androidapps, r/iosapps) 소개
∙ 기술 블로그 (CMP로 만든 앱 사례)
∙ Twitter/X 개발 과정 공유 (#buildinpublic)
∙ 지인 네트워크 초기 리뷰 확보 (20+ 리뷰)
```
