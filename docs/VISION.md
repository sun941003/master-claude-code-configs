## 미션

**"CMP 기반 1인 글로벌 앱 스튜디오"**

Kotlin/Compose Multiplatform 단일 코드베이스로 Android · iOS · Web을 동시에 지원하는 앱을 빠르게 만들고, 공유 모듈과 자동화 파이프라인으로 포트폴리오를 확장하며, 전 세계 사용자에게 서비스한다.

---

## 왜 하는가

- 본업 급여의 한계 → **부수입 채널** 구축
- 수익이 충분히 성장하면 → **독립 창업** 전환
- 개발자로서 기획부터 배포까지 직접 운영하는 경험 축적
- 78개+ 언어를 지원하는 글로벌 제품을 1인 체제로 운영할 수 있다는 것을 증명

---

## 핵심 원칙

### 1. 글로벌 우선

모든 앱은 출시일부터 78개+ 언어를 지원한다. 특정 국가 시장에 의존하지 않고, 글로벌 사용자 기반으로 수익을 분산한다.

### 2. 빠른 개발, 안정적 테스트

개발 속도는 공격적으로 유지하되, QA는 자동화로 안정성을 확보한다. 내가 직접 확인해야 할 항목은 최소화하고, 코드로 검증 가능한 것은 전부 자동화한다.

### 3. 모듈 재사용

한 번 만든 기능은 다시 만들지 않는다. IAP, Auth, 다국어, AdMob 등 공통 기능을 독립 모듈로 추출하여 신규 앱 개발 시간을 단축한다.

### 4. 자동화 우선

CI/CD, 테스트, 배포, 스토어 업데이트까지 — 자동화 가능한 모든 과정을 자동화한다. 수동 작업이 필요한 경우에도 스크립트화하여 재현 가능하게 만든다.

### 5. 사용자 수 우선

프리미엄이든 광고든, 사람이 많이 써야 수익이 난다. 무료 사용자의 경험을 해치지 않으면서 자연스러운 전환을 유도하는 UX를 설계한다.

## 목표

### 1년 (2026 Q4)

- **앱 2개 스토어 라이브** (Splitly + BetOnMe)
- **통합 MAU 10,000+**
- **월 수익 $1,000+**
- 공유 라이브러리 4개 동작 (moon-iap, moon-auth, moon-premium, moon-admob)
- CI/CD 멀티앱 파이프라인 가동
- 슬랙 알림 + 원클릭 배포 체계

### 3년 (2028 Q4)

- **앱 4-5개 운영**
- **월 수익 $10,000+**
- 오픈소스 모듈 3개+ 공개 (커뮤니티 형성)
- 1명령 → 전체 앱 빌드/테스트/배포 파이프라인
- 수익 기반 하드웨어 투자 완료 (빌드 서버)
- 독립 창업 여부 판단 가능한 데이터 확보

### 5년 (2030 Q4)

- **CMP 기반 앱 생태계** 확립
- 앱 포트폴리오 + 오픈소스 + 기술 블로그로 개발자 브랜드 구축
- 팀 확장 또는 1인 체제 유지 — 수익 규모에 따라 결정
- MoonDeveloper를 "CMP로 글로벌 앱을 만드는 사람/팀"으로 인식

## 개발 교훈 (Splitly + ureen)

| 교훈 | 출처 | 적용 |
|------|------|------|
| 다국어 지원은 타협 없이 | Splitly 성공 | 모든 앱에 78개+ 언어 필수 |
| 타겟 시장 사전 검증 필수 | ureen 실패 | 앱 기획 시 TAM/SAM 분석 포함 |
| 불분명한 요청 → 누락 발생 | Splitly 스토어 에셋 | 프롬프트 체크리스트화, 적용 검증 자동화 |
| 개발 중 기능 고도화 지속 | Splitly 성공 | 출시 후가 아닌 개발 중에도 품질 개선 |
| KMP 테스트는 desktopTest only | Splitly 루프 | allTests 금지, 5분 타임아웃, Fake 우선 |
| STUB 구현은 Phase별 추적 | Splitly Phase 12 | QA 체크리스트에 STUB 목록 관리 |
| 자동 테스팅 인프라 선행 | Splitly Phase 12 | 3계층 자동 테스트 구축 (Maestro E2E 제외 — 실기기 연결 이슈) |
